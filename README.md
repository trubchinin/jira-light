# Jira Lite

Спрощений аналог трекера задач з базовими функціями управління задачами та рольовою моделлю доступу.

## Основні можливості

-   Створення та управління проектами
-   Канбан-дошка для візуалізації процесу роботи над задачами
-   Створення та редагування задач
-   Зміна статусу задач (drag-and-drop переміщення)
-   Підтримка різних типів користувачів з розмежуванням доступу
-   Повна підтримка української мови у інтерфейсі користувача

## Ролі користувачів

-   **ADMIN** - повний доступ до всіх функцій та задач
-   **USER** - може створювати та редагувати свої задачі
-   **GUEST** - тільки перегляд задач без можливості редагування

## Тестові користувачі

| Логін | Пароль | Роль  |
| ----- | ------ | ----- |
| admin | qwerty | ADMIN |
| john  | 1234   | USER  |
| guest | guest  | GUEST |

## Технічні характеристики

-   Мова програмування: Java
-   Інтерфейс: Swing
-   Архітектура: багатошарова з розділенням на модулі (domain, service, ui)
-   Тестування: JUnit 5
-   Локалізація: повна підтримка української мови

## Архітектура проекту

Проект має багатошарову архітектуру, що забезпечує розділення відповідальності та полегшує тестування:

### Рівні архітектури:

1. **Рівень домену (jiralite-domain)**
    - Містить основні бізнес-сутності (User, Issue, Board, Project)
    - Енумерації для статусів та пріоритетів задач, ролей користувачів
    - Не залежить від інших шарів
2. **Сервісний рівень (jiralite-service)**

    - Реалізує бізнес-логіку
    - Управляє доступом до даних та обробляє їх
    - Сервіси: AuthService, BoardService
    - Використовує singleton-патерн для забезпечення глобального доступу

3. **Рівень користувацького інтерфейсу (jiralite-ui)**
    - Реалізує графічний інтерфейс користувача на Swing
    - Компоненти дошки, картки задач, панелі колонок
    - Обробка взаємодії з користувачем
    - Реалізація drag-and-drop функціональності
    - Повна локалізація інтерфейсу українською мовою

### Діаграма взаємодії модулів:

```
+------------------+          +------------------+          +------------------+
|                  |          |                  |          |                  |
| jiralite-domain  | <------- | jiralite-service | <------- |   jiralite-ui    |
|                  |          |                  |          |                  |
+------------------+          +------------------+          +------------------+
```

## Структура проекту

-   **jiralite-domain**

    -   `ua.oip.jiralite.domain` - доменні моделі (User, Issue, Board, Project, Comment)
    -   `ua.oip.jiralite.domain.enums` - енумерації (Role, Status, Priority)

-   **jiralite-service**

    -   `ua.oip.jiralite.service` - сервіси для роботи з даними
    -   `ua.oip.jiralite.repository` - репозиторії для доступу до даних (імітація роботи з БД)

-   **jiralite-ui**
    -   `ua.oip.jiralite.ui` - компоненти графічного інтерфейсу
    -   `ua.oip.jiralite.ui.components` - повторно використовувані UI компоненти
    -   `ua.oip.jiralite.ui.drag` - реалізація drag-and-drop функціональності
    -   `resources/i18n` - файли локалізації інтерфейсу (labels_uk.properties)

## Модульні та інтеграційні тести

У проекті реалізовані модульні тести для ключових компонентів:

-   `AuthServiceTest` - тестування логіки автентифікації та прав доступу
-   `BoardServiceTest` - тестування роботи з дошками та задачами
-   `ServiceIntegrationTest` - інтеграційне тестування взаємодії сервісів
-   `SimpleLoginTest` - базове тестування UI

## Локалізація

Додаток повністю локалізовано українською мовою:

-   Всі елементи інтерфейсу користувача
-   Повідомлення про помилки та підказки
-   Елементи меню та кнопки
-   Метки на формах та вікнах
-   Пошукові фільтри та системні повідомлення
-   Відображення статусів та пріоритетів задач

Локалізаційні файли знаходяться в директорії `jiralite-ui/src/main/resources/i18n`:

-   `labels_uk.properties` - файл локалізації українською мовою

## Розмежування прав

У системі реалізовано коректне розмежування прав користувачів:

1. **ADMIN** має повний доступ:

    - Може переглядати всі задачі
    - Може створювати нові задачі
    - Може редагувати будь-які задачі
    - Може змінювати статус будь-яких задач
    - Може видаляти будь-які задачі

2. **USER** має обмежений доступ:

    - Може переглядати всі задачі
    - Може створювати нові задачі
    - Може редагувати тільки свої задачі (де він виконавець)
    - Може змінювати статус тільки своїх задач
    - Може видаляти тільки свої задачі

3. **GUEST** має доступ тільки для читання:
    - Може переглядати всі задачі
    - Не може створювати нові задачі
    - Не може редагувати задачі
    - Не може змінювати статус задач
    - Не може видаляти задачі

## Встановлення та налаштування середовища розробки

### Вимоги до системи

-   JDK 11 або новіше
-   Maven 3.6 або новіше
-   Git

### Кроки для налаштування середовища

1. Клонувати репозиторій:

    ```bash
    git clone https://github.com/trubchinin/jira-light.git
    cd jira-light
    ```

2. Зібрати проект:

    ```bash
    mvn clean install
    ```

3. Імпортувати проект в IDE (IntelliJ IDEA, Eclipse):
    - Для IntelliJ IDEA: File → Open → вибрати pom.xml
    - Для Eclipse: File → Import → Maven → Existing Maven Projects

## Запуск програми

Для запуску програми виконайте:

```bash
cd jiralite-ui
java -jar target/jiralite-ui-1.0.0-SNAPSHOT.jar
```

Або для запуску з Maven:

```bash
cd jiralite-ui
mvn exec:java
```

## Запуск тестів

Для запуску всіх тестів:

```bash
mvn test
```

Для запуску тестів конкретного модуля:

```bash
cd jiralite-service
mvn test
```

## Усунення можливих проблем

1. **Помилка "Could not find or load main class"**

    - Переконайтеся, що ви перебуваєте в кореневому каталозі проекту
    - Перевірте, чи правильно зібрано проект командою `mvn clean install`

2. **Проблеми з drag-and-drop функціональністю**

    - Переконайтеся, що ви авторизовані з відповідними правами доступу
    - Користувач GUEST не може переміщувати картки

3. **Проблеми з автентифікацією**
    - Перевірте правильність введення логіну та паролю
    - Переконайтеся, що використовуєте тестові облікові записи, зазначені вище
